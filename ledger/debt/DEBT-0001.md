# DEBT-0001 — Provisional Genesis Parameters

Status: **OPEN** · Filed: 2026-07-21 · Channel: CER-3 · Priority: **medium**

## Compromise

The genesis parameters — `CAP` (kernel size cap, provisionally 12000 bytes per SC-09/B-06),
`W` (pruning window), `DL` (link-closure depth), `N_defaults` (tournament fan-out per
quadrant), and the classification thresholds — are set to **provisional constant values** at
genesis, chosen by judgment rather than by evidence.

## Reason

Genesis must pick concrete values for the system to run at all (§13.1 marks each AMENDABLE),
but PAEOS-0's own method says these should be **tournament-tuned** (PAEOS-2 §4: N is set by the
marginal value of best-of-N; PAEOS-1 §10 lists them as "open, to be set empirically"). No real
workload data exists at genesis, so evidence-based tuning is impossible on day zero. Picking
provisional values is the only way to bootstrap.

## Ideal solution

Each parameter is set (and periodically re-set) by the evolution loop from real operating data:
CAP by measured context-tax vs. constitution-completeness; N per quadrant by measured
best-of-N lift vs. verification cost; DL by measured link-closure sufficiency; thresholds by
measured misclassification rates. The Steward proposes parameter amendments backed by firing
records and incident statistics.

## Repayment conditions

Due when ≥ N complete engineering cycles of real workload data exist per parameter (enough to
estimate the relevant marginal quantity with confidence). Concretely: after the first external
workload (Sentium) has run enough Trace-B goals to populate tournament and classification
statistics. Each parameter repays independently as its data matures.

## Priority

Medium. The provisional values are safe (they are AMENDABLE and none violates an invariant),
but leaving them un-tuned indefinitely forfeits the efficiency PAEOS-0 promised and risks
silent Goodhart drift (Ω-19) if a badly-set threshold shapes selection. Not blocking for
genesis or kernel implementation.

## Estimated impact

Bounded and reversible. Worst case is suboptimal ceremony allocation (over- or under-routing
work) and a larger-than-necessary constitution tax — measurable, and correctable by ordinary
parameter amendments once data exists. No correctness risk; pure efficiency debt.

Related: none yet. If tuning reveals that a parameter *cannot* be set well within the current
schema (e.g. CAP needs per-scope budgets), that would warrant a Proposal (CER-2).
