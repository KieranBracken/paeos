# G4 — Constitution (K01–K11, D01–D02)

## Forcing clause
PAEOS-4 §4 (invariants) · K6 (every Policy carries scar + falsifier + firing record) ·
K10 (kernel size cap) · PAEOS-3 SVK-4 · producing package: **G-CONST** (Wave 0).

## Writes
Thirteen `Policy(kind=constitution, scope=kernel, loading=always)` objects:
- **K01–K11** — the eleven axiom-invariants, statements verbatim from PAEOS-4 §4.
- **D01–D02** — the two derived guarantees (crash-only agents; constraint monotonicity),
  bodies marked `derived: true` with proof citations.

Each carries, per K6: a **scar** (from `corpus/` — mostly PAEOS-3.5 review findings,
which are incurred incidents, e.g. K2←F8, K1←F10/F11, K11←F13; some Sentium-mined or
vicarious per A-18), a **falsifier**, and an initialized **firing record**.

## Inputs
`corpus/` (G-CORPUS output — the scar refs must resolve, which is why G-CORPUS
precedes G4 in dependency though both are Wave 0) · PAEOS-4 §4 text (authoritative
statements) · G-CONST scar/falsifier tables.

## Exit evidence
All 13 admitted by the validator: statements schema-valid, every scar ref resolves, every
falsifier parses, total body size ≤ CAP (K10, in bytes per B-06). This is the S0-exit
condition: **the constitution passes its own admission rules.**

## Constraint
Total ≤ CAP. If the 13 bodies exceed CAP, that is an Incident against SC-09 (CAP too
small), not license to trim statements — HALT and amend the parameter.
