# G1 (+G2) — GenesisRecord + Schemas (atomic batch)

> **Runbook, not object.** This documents the first genesis WRITE batch. The objects it
> describes are produced by Wave-0 workers and I-GEN; they do not exist yet.
> **Amendment: PAEOS-5.5 B-03 merges G1 and G2 into ONE atomic WRITE** — an object's
> `schema` field cannot pin a schema that isn't in the store, and the GenesisRecord
> itself needs a schema, so both are admitted together under the meta-schema sentinel.

## Forcing clause
PAEOS-4 §13.1 (GenesisRecord), §9.5 (schemas are kernel Policy artifacts), §2.2
(`schema: HASH` on every header) · PAEOS-5.5 B-03 (meta-schema sentinel, atomic batch) ·
PAEOS-3 §8 Q1 (the genesis record is the literal first artifact — the Grundnorm as a file).

## Writes (one atomic commit)
1. **GenesisRecord** — `schema: "sha256:meta.v1"` (sentinel). Fields (PAEOS-4 §13.1):
   - `authority: {founder_id, host_runtime}` — founder + raw Claude Code / Fable, the cross-compilation host (PAEOS-3 §1).
   - `constitutional_basis: [HASH]` — the five hashes in `../constitutional-basis.sha256` (already computed; stable while `derivation/` is frozen).
   - `scar_corpus: [HASH]` — refs into `corpus/` (G-CORPUS output).
   - `adapter_trust_roots: [(adapter_id, pubkey)]` — null + Claude Code Ed25519 public keys (SC-03).
   - `parameters: {CAP, W, DL, N_defaults, thresholds}` — each marked AMENDABLE (CAP=12000 provisional per SC-09, **restated in bytes per B-06**).
   - `assumptions:` — single-founder (H1), parchment clause (H2), NL-as-policy-encoding + falsifier (H5), token-economics scar of K10 + falsifier (H6) — per A-19.
   - `expiry_condition:` — the PAEOS-3 §6 self-hosting criterion, verbatim (this record names its own death).
2. **Schema Policy artifacts** — the six object schemas (Artifact, Goal, Evidence, Policy) as amended: 4 primitives not 6 (A-02/A-03), plus the missing schemas from PAEOS-5.5 H-04 (SpawnRecord, DispatchRecord, FiringEvent, Check, ref-manifest formats). Each is a `Policy(kind=schema-artifact)` validated by the meta-schema.

## Inputs
`../constitutional-basis.sha256` (ready) · `corpus/index.json` (G-CORPUS) ·
adapter public keys (from I-NULL / I-CC key generation) · F-SCHEMA schema definitions
(the authoritative field lists; the reissued F-SCHEMA package per PAEOS-5.5 H-04).

## Exit evidence
The batch commits and is admissible under the meta-schema. No store-loaded schema exists
before this batch, so it is the only batch the meta-schema validates; everything after is
validated by these schemas (breaks the validator↔store genesis cycle — PAEOS-5.5 §02 F/cycle-1).

## Preconditions (gate)
Three kernel amendments resolved (B-01, B-02, B-06); F-SCHEMA reissued; adapter keys exist.
